安装与启动

  1. 克隆项目：git clone <your-repo-url> NexBlog
  2. 启动后端：cd backend → ./gradlew bootRun
  3. 启动前端：cd front → npm install → npm run dev
  4. 初始化管理员：访问 /studio/login 按页面指引完成首次配置
  5. 默认地址：前端 http://localhost:5173，后端 http://localhost:8080

一键发布流程

Step 1 · 生成静态包

在全局设置页点击“生成静态包”，系统会导出数据并完成静态资源构建。

Step 2 · 本地预览验证

点击“打开本地预览”，确认页面与资源无异常后再执行发布。

Step 3 · 发布到 GitHub

点击“发布到 GitHub”后自动推送构建产物到目标分支并上线。当前链路已验证可用。

常见问题

这个门户页是否依赖 Vue 项目？

不依赖。它是单独的 HTML 文件，可直接双击打开或单独部署到 GitHub Pages。

门户页和博客主站应该如何分仓？

建议门户页独立仓库，主站继续使用现有发布链路。这样官网信息与业务代码可以独立迭代。

发布按钮成功后仍有问题怎么办？

先看预览地址和浏览器网络面板，再核对静态包目录与资源前缀。该项目已具备快速定位链路。

准备上线门户页？

将当前文件放到单独仓库根目录并开启 GitHub Pages，即可获得一个正式的项目官网入口。
